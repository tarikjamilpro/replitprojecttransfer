Professional Finance Calculators - Loan EMI Calculator Application
Project Overview
Build a modern, responsive Finance Calculators web application with a fully functional Loan EMI Calculator using Next.js 16, React 19, and TypeScript. The application should provide accurate financial calculations with visual data representation.

Core Requirements
1. Input Fields - Loan Parameters
Implement three primary input controls with validation:

A. Loan Amount (Principal)

Input type: Number with currency formatting
Label: "Loan Amount" or "Principal Amount"
Placeholder: "e.g., 500000"
Validation: Minimum ₹10,000, Maximum ₹1 Crore (adjustable)
Format: Display with commas (₹5,00,000)
B. Interest Rate (Annual)

Input type: Number (decimal allowed)
Label: "Annual Interest Rate (%)"
Placeholder: "e.g., 8.5"
Validation: Minimum 1%, Maximum 30%
Step: 0.1 (allow one decimal place)
C. Loan Tenure

Provide toggle/tabs for Years OR Months selection
Input type: Number
Label: "Loan Tenure"
Years: Minimum 1, Maximum 30
Months: Minimum 1, Maximum 360
Default unit: Years
Additional Enhancement:

Add range sliders alongside input fields for better UX
Sync slider and input field values in real-time
Show min/max values on sliders

Design Guidelines
Color Scheme:

Choose 3-5 colors total:

1 primary brand color (for EMI amount, CTA buttons)
2-3 neutrals (backgrounds, text, borders)
1-2 accent colors (for interest, charts)
Avoid purple/violet unless explicitly requested
Ensure high contrast for readability
Typography:

Maximum 2 font families
Use font-sans for body text
Large, bold numbers for financial amounts
Clear hierarchy: h1 for page title, h2 for sections, h3 for cards
Layout Structure:

Mobile-first design
Use Flexbox for most layouts
CSS Grid only for results grid
Generous spacing between sections
Card-based design for inputs and outputs
Visual Hierarchy:

EMI amount should be the most prominent element
Inputs section → Calculation button (optional) → Results section → Chart
Use visual separators (borders, shadows, backgrounds)
