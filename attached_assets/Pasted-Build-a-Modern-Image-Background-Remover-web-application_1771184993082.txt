Build a Modern Image Background Remover web application using React and Tailwind CSS. This tool allows users to upload images (PNG or JPG) and remove their backgrounds using AI-powered processing, all handled locally in the browser without requiring external API keys. The app provides a seamless, professional experience with drag-and-drop uploads, live previews, and easy downloads, making it ideal for designers, marketers, and casual users. Focus on efficiency, responsiveness, and a clean UI to ensure fast processing and a great user experience.

Core Requirements:
Main Features:

Image Upload: A prominent drag-and-drop zone or file upload button that accepts PNG and JPG files. Display visual feedback during drag (e.g., border highlight) and show a thumbnail preview of the uploaded image immediately.
Background Removal Processing: Integrate the "@imgly/background-removal" library (or a similar open-source alternative like "remove.bg" client-side SDK if available) to process the image locally. The processing should run asynchronously in the browser, outputting a transparent PNG with the background removed.
Previews: Show a side-by-side "Before" and "After" view:
"Before": Displays the original uploaded image.
"After": Updates dynamically once processing completes, showing the background-removed version. If processing fails, display an error message in the "After" section.
Progress Indication: While processing, show a progress bar (e.g., animated bar filling up) or a spinner with text like "Removing background, please wait...". Disable interactions during processing to prevent multiple uploads.
Download and Reset:
A "Download" button that becomes enabled after successful processing, allowing users to download the transparent PNG.
A "Reset" button to clear the current image and start over, resetting the previews and inputs.
Error Handling: Display user-friendly error messages for unsupported formats, file size limits (e.g., max 10MB), or processing failures (e.g., "Unable to process image. Please try a different file."). Use toast notifications or inline alerts.
UI/UX Design:

Layout: Centered, clean design with the upload area at the top, followed by the side-by-side previews below. On mobile, stack the previews vertically. Use ample white space and subtle shadows for depth.
Styling with Tailwind CSS:
Professional look: Soft grays, whites, and accent colors (e.g., blue for buttons). Use Tailwind's utilities for borders, shadows, and transitions.
Upload Area: Large, dashed border box with an icon (e.g., cloud upload) and text like "Drag & drop your image here or click to browse". Highlight on hover/drag.
Previews: Equal-width containers with labels ("Before" and "After"). Images should be responsive, maintaining aspect ratio, with a max height for consistency.
Buttons: Rounded, prominent "Download" (green or blue) and "Reset" (gray) buttons with hover effects. Disable states during processing.
Animations: Smooth fade-ins for previews, progress bar animations, and button transitions. Use Tailwind's transition utilities.
Accessibility: Keyboard navigation for uploads and buttons. ARIA labels for the drag-drop area, alt text for images, and screen-reader announcements for progress and errors. High-contrast support.
Responsiveness: Fully mobile-friendly—upload area and previews adapt to small screens. Touch-friendly button sizes and drag-drop on mobile devices.
Data Management:

File Handling: Use the browser's File API to read uploaded images. Convert to a format compatible with the background removal library (e.g., base64 or blob). Validate file type and size on upload.
Processing Logic: Run the AI processing in a web worker if possible to avoid blocking the UI. Store the original and processed images in state for previews.
State Management: Use React hooks (useState, useEffect) to track upload status, processing progress, and results. Clear state on reset.
No External Dependencies Beyond Library: Rely on "@imgly/background-removal" for core functionality; ensure it's loaded efficiently.
Technical Implementation:

React Structure: Functional components with hooks for state and effects.
Component Breakdown:
App: Main container managing overall state and layout.
UploadArea: Handles drag-and-drop and file selection, with validation.
PreviewSection: Side-by-side container for "Before" and "After" images.
ProgressIndicator: Component for spinner/bar during processing.
ActionButtons: "Download" and "Reset" buttons with conditional rendering.
Performance: Optimize image loading and processing to keep it under 10-20 seconds for typical images. Use lazy loading for previews and memoization to prevent unnecessary re-renders.
Dependencies: React, Tailwind CSS, and "@imgly/background-removal". Install via npm and follow the library's documentation for integration.
Browser Compatibility: Ensure support for modern browsers with WebAssembly (required for the library). Provide fallbacks for unsupported environments.
Bonus Features (Optional for Enhancement):

Batch Processing: Allow uploading multiple images and process them sequentially, with a queue indicator.
Image Editing: Add basic post-processing options like cropping or adjusting transparency levels.
Themes: Light/dark mode toggle using Tailwind's dark utilities.
Export Options: Allow downloading in different formats (e.g., JPG with white background) or resizing.
Analytics/Feedback: Track usage (locally) and add a "Rate this tool" section.
Integration: Placeholder for future API integration if users want cloud-based processing.
Example Implementation Notes:
Library Integration: Install "@imgly/background-removal" and use its removeBackground function. Example: const result = await removeBackground(imageBlob); then display the result as a transparent PNG.
Sample State Structure: { originalImage: null, processedImage: null, isProcessing: false, error: null }
File Validation: Check file.type for "image/png" or "image/jpeg" and file.size < 10 * 1024 * 1024.
